# フリーランス向け 税金シミュレータ 仕様書 v1.1

作成日: 2026-07-02 / 更新: v1.1で解説コンテンツを追加 / ステータス: ドラフト(要レビュー)

## 1. 目的・コンセプト

フリーランス・個人事業主が、年間売上と経費を入力するだけで、複数税目を横断した税・社会保険料の内訳と手取り額を一括で試算できるWebアプリ。さらに「法人成りした場合」との総負担比較まで行えることを差別化ポイントとする。

- 想定ユーザー: フリーランス・個人事業主(税務の専門知識なしを前提)
- 提供価値: (1) 手取りの見える化 (2) 消費税の課税方式比較 (3) 法人成り判断の目安
- 位置づけ: あくまで概算シミュレーション。申告・納税額の確定用途ではない(免責表示必須)

## 2. スコープ

### 2.1 対象(In Scope)

| # | 項目 | 内容 |
|---|------|------|
| 1 | 所得税+復興特別所得税 | 累進税率(5〜45%)+復興特別2.1% |
| 2 | 住民税 | 所得割10%+均等割(森林環境税含む) |
| 3 | 個人事業税 | 業種別税率3〜5%、事業主控除290万円 |
| 4 | 消費税 | 課税事業者判定、インボイス登録、本則/簡易/2割特例の比較 |
| 5 | 国民健康保険・国民年金 | 手取り計算のための概算(税ではない旨明示) |
| 6 | 法人成り比較 | 個人事業継続 vs 法人化の総負担比較(簡易モデル) |
| 7 | 解説コンテンツ | 各税目の計算方法・用語(小規模企業共済等)の学習ページ |
| 8 | 65万円控除チェック | 質問に答えると適用可能な青色申告特別控除(65/55/10万・白色)を判定 |

### 2.2 対象外(Out of Scope)

- 給与所得・不動産所得・譲渡所得など事業所得以外の所得(v2候補)
- 予定納税、延滞税、住宅ローン控除・ふるさと納税上限などの税額控除詳細
- 自治体別の国保料率の完全網羅(概算モデルで代替)
- 年分切替UI(内部設計は対応するが、UIは令和8年分固定でリリース)

## 3. 入力仕様

### 3.1 基本入力

- 年間売上(税込/税抜の別を選択)
- 必要経費: 金額指定 or 経費率(%)指定の切替
- 申告区分: 青色65万(e-Tax・電子帳簿) / 青色55万 / 青色10万 / 白色

### 3.2 所得控除(任意入力、初期値0)

- 社会保険料控除(国保・国民年金は自動計算値を自動反映、手入力上書き可)
- iDeCo・小規模企業共済(小規模企業共済等掛金控除)
- 配偶者控除・扶養控除(人数入力による簡易対応)
- 生命保険料控除・医療費控除(金額直接入力)

### 3.3 消費税関連

- 基準期間(前々年)の課税売上高
- インボイス発行事業者の登録有無
- 課税仕入(経費のうち消費税がかかるものの割合、既定値を用意)
- 簡易課税の事業区分(第1種〜第6種、みなし仕入率90〜40%)

### 3.4 法人成り比較用

- 想定役員報酬(月額) — スライダーで変更し即時再計算
- 法人側の追加コスト(税理士報酬等)の概算入力(任意)

### 3.5 国保用

- 居住自治体の料率プリセット(主要都市数件)+「全国標準モデル」既定値
- 年齢(40歳以上で介護分加算)、世帯加入人数

## 4. 計算仕様

### 4.1 計算フロー(個人)

```
売上 − 経費 = 事業所得(青色特別控除前)
− 青色申告特別控除 = 事業所得(合計所得金額)
− 所得控除(基礎控除・社保控除ほか) = 課税所得 ※所得税と住民税で別計算
課税所得 × 累進税率 − 控除額 = 所得税 → ×102.1% = 所得税+復興特別所得税
住民税 = 課税所得(住民税ベース) × 10% + 均等割
個人事業税 = (事業所得 ※青色特別控除は戻す − 290万) × 業種別税率
```

### 4.2 税制パラメータの外部化(重要)

税率表・控除額・判定閾値はすべてコードから分離した年分別パラメータオブジェクト(JSON)として保持する。理由:

- 令和8年分の基礎控除は「本則62万円+時限特例」の二層構造(合計所得489万円以下: 104万円 / 489万超〜655万以下: 67万円 / 655万超〜2,350万以下: 62万円)であり、令和7年分とも令和10年分以降とも金額が異なる
- 住民税の基礎控除は改正されておらず43万円のままで、所得税と非対称
- 毎年の税制改正への追従をパラメータ差し替えだけで行えるようにする

パラメータの正本は実装時に国税庁・総務省の一次資料で確定させる(高所得帯の基礎控除逓減、均等割額、国民年金保険料月額、国保賦課限度額など)。

### 4.3 消費税

1. 課税事業者判定: 基準期間の課税売上高 > 1,000万円、またはインボイス登録あり
2. 課税事業者の場合、以下3方式を並列計算して比較表示:
   - 本則課税: 売上消費税 − 仕入消費税
   - 簡易課税: 売上消費税 × (1 − みなし仕入率)
   - 2割特例: 売上消費税 × 20%(適用可能期間の判定ロジックを持つ。インボイス登録を機に課税事業者となった者限定・時限措置のため、対象年分で適用可否を自動判定)
3. 適用不可の方式はグレーアウトし理由を表示

### 4.4 国保・国民年金(概算)

- 国民年金: 定額(月額×12、パラメータ管理)。付加年金・免除は対象外
- 国保: 所得割+均等割(+平等割)の簡易モデル。医療分+後期高齢者支援金分(+40歳以上は介護分)、賦課限度額でキャップ
- 前年所得ベースで賦課される制度だが、シミュレータでは「入力年の所得が継続する定常状態」と仮定して計算(この仮定を画面に明記)

### 4.5 法人成り比較(簡易モデル)

- 法人側: 課税所得 = 事業利益 − 役員報酬 − 法人負担社会保険料。法人税(中小軽減税率考慮)+地方法人税+法人住民税(均等割含む)+法人事業税を概算
- 個人側: 役員報酬を給与所得として所得税・住民税を計算(給与所得控除適用)、協会けんぽ+厚生年金(本人負担分)
- 出力: 「個人事業のまま」vs「法人成り」の(税+社会保険料)総負担と世帯手取りを並列表示。役員報酬スライダーで損益分岐を探索できる
- 精度限界(退職金活用・消費税免税期間・配偶者役員などは未考慮)を明記

### 4.6 端数処理

- 課税所得: 1,000円未満切捨て
- 確定税額: 100円未満切捨て(所得税)
- その他税目も法令の丸めルールに準拠(実装時に税目別に確定)

## 5. 出力(結果画面)

- 自由に使えるお金(最重要KPIとして最大表示): 売上 − 経費 − 税金 − 社会保険料。詳細設定でiDeCo・小規模企業共済の掛金を入力した場合は、積立額(自由に使えないお金)も差し引いた金額を表示する
- 差引フローのリスト表示: 「年間売上 → −経費 → −税金 → −社会保険料 → 残り」の順に1行ずつ表示。税金・社会保険料の行はタップで税目別の内訳に展開
- 消費税3方式の比較、法人成り比較、実効負担率は別画面(リンクで遷移)

## 6. 解説コンテンツ(学習ページ)

### 6.1 構成

シミュレータと同一SPA内に「しくみを学ぶ」タブを設け、以下の3層で構成する。

1. **税目別解説**: 所得税・住民税・個人事業税・消費税・国保/国民年金の各ページ。「誰に・何に対して・どう計算されるか」を計算式のフロー図解付きでステップ解説する
2. **用語集**: 小規模企業共済、iDeCo、青色申告特別控除、基礎控除、所得控除と税額控除の違い、合計所得金額と収入の違い、インボイス制度、基準期間、みなし仕入率、事業主控除、賦課限度額など。カテゴリ別一覧+キーワード検索
3. **入力値連動の計算例**: シミュレータに入力済みの場合、解説内の計算例をユーザー自身の数字で展開して表示する(未入力時はモデルケースのサンプル値)。「一般論の説明」と「自分ごと化」を橋渡しする本アプリならではの差別化要素

### 6.2 導線設計

- 入力フォームの各項目に「?」アイコン → ポップオーバーで1〜2文の要約 → 「詳しく見る」で該当解説ページへ遷移
- 結果画面の各税目カード・各控除項目から該当解説へ直接リンク
- 解説ページから「この数字でシミュレーションする」で入力画面へ戻れる双方向導線

### 6.3 執筆・管理方針

- 専門知識ゼロを前提にした平易な文章。専門用語には必ず用語集リンクを付す
- 解説内に登場する税率・控除額は、計算エンジンと同じ税制パラメータ(4.2)から参照して描画し、本文へのハードコードを禁止する(改正時の記述漏れ・不整合を構造的に防ぐ)
- 「一般的な情報提供であり、個別の税務相談は税理士へ」という位置づけを解説ページにも明記

### 6.4 65万円控除チェック(判定ページ)

- 独立した別ページとして提供。5問のはい/いいえで、適用可能な青色申告特別控除を判定する
  1. 青色申告承認申請書を提出済みか(いいえ → 白色。提出期限の解説へリンク)
  2. 複式簿記で記帳しているか(いいえ → 10万円控除)
  3. 貸借対照表・損益計算書を添付できるか(いいえ → 10万円控除)
  4. 期限内申告か(いいえ → 10万円控除)
  5. e-Tax申告または優良な電子帳簿保存か(いいえ → 55万円、はい → 65万円)
- 判定結果とあわせて「あと何をすれば上位の控除になるか」と、その場合の節税額(シミュレータの入力値で計算)を表示
- 「この区分をシミュレータに反映」ボタンで申告区分に1タップ反映
- 判定ロジックの根拠数値も税制パラメータ(4.2)から参照する

## 7. 非機能要件

- UI方針(段階的開示): 初期表示は最小3入力(売上・経費・申告区分)+その直下の「詳細設定」(折りたたみ)+自由に使えるお金、の縦一列構成。控除・消費税・国保等の設定は詳細設定に格納し、既定値で即計算する。装飾・色数は最小限とし、消費税比較・法人成り比較・解説は別画面に分離する

- クライアント完結: 入力データを一切外部送信しない(プライバシー訴求として明記)
- レスポンシブ: スマホ幅(375px)〜デスクトップ対応
- 即時再計算: 入力変更でリアルタイム更新
- 計算エンジンは純関数として分離し、税制パラメータを注入する設計(テスト容易性)
- 免責表示: 「本シミュレーションは概算です。実際の税額は申告内容・自治体により異なります」を常時フッター表示

## 8. 技術構成(案)

- アーキテクチャ: 完全静的SPA(バックエンドなし)。計算・データ保持はすべてブラウザ内で完結し、外部通信は行わない。税制パラメータJSONはアプリに同梱。税制改正時はパラメータ差し替え+再デプロイで対応

- React SPA(単一ファイル、Claude Artifactとして動作)
- グラフ: Recharts
- 状態管理: React state(メモリ内のみ。ブラウザストレージは使用しない)
- 構成: `taxParams(年分別JSON)` / `calcEngine(純関数群)` / `UI(入力フォーム+結果ダッシュボード)`

## 9. 未決事項(実装前に確定)

3. 個人事業税の業種プリセット(既定は第1種5%、非課税業種の選択肢を用意するか)
