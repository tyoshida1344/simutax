# CLAUDE.md — 税金シミュレータ 実装ガイド

このリポジトリはフリーランス向け税金シミュレータ(完全静的SPA)。詳細要件は同梱の「税金シミュレータ仕様書_v1.md」を必ず先に読むこと。本ファイルは実装時の技術判断・作法をまとめた補足。

## 技術スタック

- Vite + React + TypeScript
- グラフ不要(現デザインは罫線リストのみ)。将来必要になったらRechartsを追加
- テスト: Vitest(計算エンジンは必須でユニットテストを書く)
- バックエンドなし・外部通信なし。静的ホスティング(GitHub Pages / Cloudflare Pages)にデプロイ可能な構成にする

## 設計原則(必ず守る)

1. **税制パラメータのハードコード禁止**。税率・控除額・閾値はすべて`params/`のJSONに置き、エンジンには引数で注入する。解説文中に表示する数値も同じパラメータから描画する
2. **計算エンジンは純関数**。入力(数値+パラメータ)→出力(税額内訳)のみ。React・DOMに依存させない
3. **所得税と住民税の控除額は非対称**(例: 基礎控除は所得税が二層構造、住民税は43万円固定)。控除テーブルを税目別に分けて持つ
4. **端数処理**は仕様書4.6に従い、税目ごとにテストで担保する
5. **localStorage等のブラウザ保存は使わない**(状態はメモリ内のみ)。将来の公開版で入力保存を入れる場合は別途判断

## 税制パラメータについての重要な注意

- 対象は令和8年(2026年)分。仕様書4.2に記載の基礎控除の二層構造(本則+時限特例)を正しく実装すること
- **実装前に必ず国税庁・総務省・財務省の一次資料でパラメータの確定値を検証すること**(基礎控除の逓減区分、住民税均等割+森林環境税、国民年金保険料月額、国保の料率モデルと賦課限度額、2割特例の適用期限判定)。二次情報のブログ記事を鵜呑みにしない
- パラメータJSONには出典URLと確認日をメタデータとして残す

## UIデザイン指針(モックアップで合意済み)

- 縦一列・1カラム(max-width 440px程度、中央寄せ)。白基調フラット、装飾と色は最小限
- 画面構成(上から): 3入力(年間売上/必要経費/申告区分) → 詳細設定(折りたたみ) → 「自由に使えるお金」(最大の数字・40px級) → 差引フローのリスト(売上→−経費→−税金→−社会保険料→残り。税金/社保の行はタップで税目別に展開) → 「法人成りと比較」「計算のしくみを見る」リンク → 免責文
- 「自由に使えるお金」はiDeCo・小規模企業共済の掛金入力時、積立額も差し引く(仕様書5参照)
- 入力変更で即時再計算。計算ボタンは置かない
- 免責文「概算です。実際の税額は申告内容・自治体により異なります」を常時表示
