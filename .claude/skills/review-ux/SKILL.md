# Review UX — UX レビュー

変更が利用者にとって使いやすいかを確認し、改善点を指摘する。

## 対象の特定

1. 変更ファイルを取得する:

```sh
git diff main --name-only
```

2. 変更ファイルの全体を読む（UI コードは diff だけでは全体像が掴めない）
3. 関連するコンポーネント・CSS を読み、画面上での見え方・動き方を把握する

## 確認観点

### 操作フィードバック

- 入力変更時に計算結果がリアルタイムで更新されるか
- 金額の表示フォーマット（カンマ区切り、「円」表記）が統一されているか
- 折りたたみ・展開の開閉状態が視覚的に明確か

### エラーハンドリング

- 不正な入力（負の値、非数値、極端に大きい値）に対して適切にハンドリングされるか
- 入力が空の場合にデフォルト値（0）で計算されるか、エラーになるか
- 経費が売上を超える場合の表示が適切か

### キーボード操作

- 入力フォームのフォーカス順序が視覚的な配置と一致しているか
- 数値入力で `inputMode="numeric"` が設定されているか（モバイルでテンキーが出るか）
- Tab キーで全入力項目を移動できるか

### 視覚的一貫性

- 既存のコンポーネントと見た目・操作感が揃っているか（フォントサイズ、余白、角丸）
- CSS カスタムプロパティ（`var(--border)`, `var(--text-secondary)` 等）を使っているか
- 「自由に使えるお金」の表示サイズ・位置が仕様（40px級、中央）に沿っているか

### レスポンシブ対応

- スマホ幅（375px）で横はみ出し・重なりがないか
- 数値入力・セレクトがモバイルで操作しやすいサイズか
- 差引フローの行が狭い画面でも読みやすいか

### アクセシビリティ

- `label` と `input` が適切に紐付いているか（`htmlFor` / `id`）
- ボタンやアイコンに適切なテキストラベルがあるか
- 色だけに頼らない情報伝達になっているか（色覚多様性への配慮）
- フォーカス時の視覚的インジケータ（アウトライン等）があるか

### 免責表示

- 免責文「概算です。実際の税額は申告内容・自治体により異なります」が常時表示されているか
- 「入力データは外部に送信されません」の記載があるか

## 出力

指摘事項を以下の形式で報告する:

```
### UX レビュー結果

**操作フィードバック**
- [ ] `ファイル名:行番号` — 指摘内容（改善案）

**エラーハンドリング**
- [ ] ...

**キーボード操作**
- [ ] ...

**視覚的一貫性**
- [ ] ...

**レスポンシブ対応**
- [ ] ...

**アクセシビリティ**
- [ ] ...

**免責表示**
- [ ] ...

該当カテゴリに指摘がなければそのカテゴリごと省略する。
全て指摘なしの場合: 「UX 上の問題は見つかりませんでした」
```

修正が求められている場合（issue-handler 経由）は、報告後に指摘箇所を修正する。
